# Workflow: Crystallize

## Objective

Extract reusable knowledge from the current session and propose long-term memory
updates.

## Save Candidates

- important decisions
- durable user preferences
- open tasks
- system design changes
- test results
- reusable operating lessons

## Procedure

1. List candidate memory updates.
2. Classify each as `memory`, `permanent-note`, `task`, or `discard`.
3. Ask for confirmation before writing.
4. Update `99_System/Memory/INDEX.md` if startup context changes.

## Safety

- Do not silently overwrite permanent notes.
- Do not store transient discussion noise.
- Prefer concise memory over full transcript.
